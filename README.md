# Реализация схемы подписи Меркла на основе хеш-функции Стрибог

Данный проект представляет собой реализацию криптографической схемы цифровой подписи Меркла (Merkle Signature Scheme) с использованием отечественного стандарта хеширования Стрибог (ГОСТ Р 34.11-2012).

## 📋 Описание

Схема подписи Меркла позволяет подписывать ограниченное количество сообщений (N = 2^n) с использованием одного корневого публичного ключа. В основе лежит бинарное дерево Меркла, листьями которого являются публичные ключи одноразовых подписей Лэмпорта.

### Основные компоненты

- **LamportSignature** — реализация одноразовой подписи Лэмпорта (256 бит)
- **MerkleTree** — шаблонная реализация дерева Меркла
- **MerkleTreeGost** — специализация дерева Меркла для хеш-функции Стрибог
- **MerkleSignature** — основная реализация схемы подписи Меркла
- **GOST** — реализация хеш-функции Стрибог (256 и 512 бит)
- **CryptoRandomGenerator** — криптографический генератор случайных чисел

## 🚀 Возможности

- Генерация ключевой пары Меркла (приватный + публичный ключ)
- Подписание сообщений с использованием одноразовых ключей Лэмпорта
- Проверка подписей с использованием только публичного ключа
- Сохранение и загрузка ключей из файлов
- Сериализация/десериализация подписей
- Поддержка хешей длиной 256 и 512 бит

💡 Пример использования
```c++
// Создание подписи Меркла (высота 3 = 8 подписей)
MerkleSignature merkleSign(3, 256);

// Сохранение ключей
merkleSign.savePrivateKey("private.key");
merkleSign.savePublicKey("public.key");

// Подписание сообщения
std::string message = "Hello, World!";
auto msgBytes = std::vector<uint8_t>(message.begin(), message.end());
auto signature = merkleSign.sign(msgBytes, 0);

// Проверка подписи
bool valid = MerkleSignature::verify(msgBytes, signature, 0, "public.key");
```

```bash
#Публичный ключ (MERKLEPK)
Магическое число (8 байт)

Корневой хеш (vector<uint8_t>)

#Подпись
Индекс сообщения (size_t)

Подпись Лэмпорта (256 × 32 байт)

Публичный ключ Лэмпорта (2 × 256 × 32 байт)

Доказательство Меркла (переменная длина)
```
### ⚠️ Ограничения
Количество подписей ограничено и фиксировано при генерации ключа

Каждый индекс может быть использован только один раз

Размер подписи увеличивается с высотой дерева
